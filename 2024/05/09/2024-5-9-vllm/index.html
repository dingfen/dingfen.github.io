Paged Attention

原理

Paged Attention 是 vllm 用来解决 KV cache 利用率不高的问题的优化技术。它仿照了操作系统中经典的分页技术（paging）。Paged Attention 通过切分一个 sequence 序列中的 KV cache 到 KV blocks 的方法，允许在非连续的内存空间存储连续的 key 和 value。每一个 KV block 会存储一定数量 tokens 的 K，V 向量，这样就将原本 KV cache 切分成一块块 KV blocks，如下图所示：

相应的，KV cache 的计算公式会被改写。

参数与数据结构

参数解释与 llama3-8b 的相关数据情况：

参数 llama3-8b 中的值 描述
num_seq 4(batch_size) 该推理中 sequence 个数
num_head 32 Query 的 head 个数
num_kv_heads 8 Key、Value 的 head 个数
hidden_size 4096 输入的嵌入张量维度
head_size 128 每个注意力头的维度大小
x 2(FP16) 数据类型的字节数
scaling 128^-0.5 注意力公式中的scale值
